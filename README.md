KhedmaNow is a **local services booking platform prototype**. It demonstrates the workflow between customers seeking services and providers managing those requests. 
This project focuses on the **Frontend User Experience**, using JSON data to simulate database interactions and state management.

---

## üöÄ Features

### 1- Dynamic Service Browsing
- Users can navigate through various service categories.
- Detailed views for specific services populated by JSON data.
- **Why it matters:** Clear navigation structure for finding services.

### 2- Role-Based Dashboards
- **Customer View:** Focused on browsing services and tracking personal requests.
- **Provider View:** Focused on managing incoming orders and publishing offers.
- **Admin View:** Focused on user management and platform statistics.
- **Why it matters:** Customizes the UI for different user needs.

### 3- Service Booking Flow
- **Booking Form:** Collects user inputs (Date, Address, Service Details).
- **Request Submission:** Simulates sending an order to the provider.
- **Why it matters:** Demonstrates the core data collection process.

### 4- Request Management (Provider)
- Providers can view a list of all client requests.
- Ability to toggle request status (Pending / Accepted / Rejected).
- **Why it matters:** Shows the operational workflow for service companies.

### 5- Offers Management
- Providers can use a form to add new "Hot Offers."
- New offers are dynamically rendered on the Offers page.
- **Why it matters:** Allows providers to update content dynamically.

### 6- User Dashboard & Notifications
- Clients can track the status of their bookings in a dedicated history view.
- specific Notification page to view system alerts.
- **Why it matters:** Keeps the user informed about their request state.
